# CI/CD Integration

This framework uses GitHub Actions for Continuous Integration.

## Pipeline Configuration (`.github/workflows/ui-tests.yml`)
The workflow automatically triggers on:
- Pushes to `main` or `master` branches
- Pull Requests against `main` or `master`

### Pipeline Steps:
1. **Checkout**: Pulls the repository code.
2. **Setup Node**: Configures the Node environment (LTS).
3. **Install Dependencies**: Runs `npm ci` for a clean install.
4. **Install Browsers**: Installs Playwright browser binaries (Chromium, Firefox, WebKit).
5. **Run Tests**: Executes the full Playwright UI test suite across all configured browsers.
6. **Upload Artifacts**: Always uploads the `playwright-report/` directory regardless of test outcome. This report contains the HTML report, and any screenshots/videos/traces generated by test failures.
7. **Publish Test Report**: Auto-parses the `results.xml` JUnit file to display pass/fail metrics directly in the GitHub PR/Commit UI.

## Local equivalent
To run the exact command the CI server runs, execute `npx playwright test` locally.
6. **Report Publishing**: Uploads the HTML report (`playwright-report/`) to the GitHub workflow artifacts for review upon success or failure.
